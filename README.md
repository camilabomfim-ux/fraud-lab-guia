# Fraud Lab — Guia da Operação

Guia interativo de scorecards de qualidade do Fraud Quality Team.

## 🔗 Acesso

Após publicar no GitHub Pages, o guia ficará disponível em:
```
https://<seu-usuario>.github.io/<nome-do-repositorio>
```

## 📁 Estrutura

```
index.html   → o guia completo (tudo em um único arquivo)
README.md    → este arquivo
```

## 🚀 Como publicar

1. Crie um repositório no GitHub (pode ser privado ou público)
2. Faça upload do `index.html` e do `README.md`
3. Vá em **Settings → Pages**
4. Em **Source**, selecione **Deploy from a branch**
5. Escolha a branch `main` e a pasta `/ (root)`
6. Clique em **Save**
7. Aguarde ~1 minuto — o link aparecerá no topo da página de Settings → Pages

## 🔄 Para atualizar o guia

Basta substituir o arquivo `index.html` por uma nova versão.
O GitHub Pages publica automaticamente em ~1 minuto após o upload.

## 🔒 Controle de acesso

- **Repositório público** → qualquer pessoa com o link consegue acessar
- **Repositório privado** → apenas membros da organização conseguem acessar
  (requer GitHub Team ou Enterprise para GitHub Pages em repositórios privados)

Para compartilhar apenas com o time sem tornar público, a alternativa mais simples
é usar o Google Drive para hospedar o `index.html`.
